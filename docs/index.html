A wallpaper picker. Nothing more.

Most wallpaper apps want to be a service. They sync. They account. They recommend. Mural reads a folder.

That is the architecture. A 5 MB Tauri binary. Sits in the menu bar. Opens a popover when you press the hotkey. Applies a wallpaper when you click one. No cloud. No inbox. No premium tier.

You bring the images. Drop them in ~/Mural, or point Mural at a public GitHub repo. Mural watches the folder, shallow-clones the repo, caches thumbnails by content hash. Then it stays out of the way.

Two source kinds.

Local folders and any GitHub repo. Local sources watch via notify. GitHub sources shallow-clone via libgit2 and re-sync on the interval you set, with a git ls-remote HEAD check that skips the fetch when the SHA hasn't moved.

# ~/Library/Application Support/Mural/config.toml

[[sources]]
kind = "github"
url  = "https://github.com/dharmx/walls"
ref  = "main"
path = "macOS"
sync_interval_hours = 24

[[sources]]
kind = "local"
path = "~/Mural"

The mechanic.

Drop wallpapers in ~/Mural. Press ⌘⇧W from any app, any Space, any Stage Manager pile. Click a thumbnail. The wallpaper applies in the same frame. Esc dismisses. You go back to work.

❦

Honest beta notes.

It's a 0.1. Some edges are still rough.

  • Unsigned. Gatekeeper warns the first time. Right-click → Open.
  • x86_64 DMG only. Apple Silicon native build comes next.
  • Lock-screen wallpaper is best-effort. Admin osascript copy.
  • Video wallpapers fall through to a static apply. Real overlay layer is on the list.
  • Per-Space targeting and CoreLocation sunrise/sunset are stubs.
  • HEIC decoding shells out to sips. No libheif bundled.
